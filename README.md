## 1. 项目概述

### 1.1 项目背景
本项目旨在开发一款功能完整的外卖软件，覆盖用户点餐、商家接单、骑手配送、后台管理及数据智能分析等核心业务。通过现代化前后端分离架构与大数据技术，实现对业务数据的实时/离线计算、数据同步与调度，为运营决策提供数据支撑。

### 1.2 项目目标
- **用户端**：提供流畅的点餐体验，包括浏览商家、下单、支付、订单追踪等。
- **商家端**：提供菜品管理、接单、营业数据查看等功能。
- **骑手端**：实现接单、取餐、送单、收益统计。
- **管理后台**：支持平台对用户、商家、骑手、订单等进行管理。
- **大数据平台**：构建数据仓库，完成实时指标计算、离线数据分析、报表生成。
- **数据同步与调度**：实现业务数据库与大数据平台的高效数据同步及任务编排。

---

## 2. 技术架构

- **前端**：Vue.js + Axios
- **后端**：Spring Boot + MyBatis + MySQL
- **大数据计算**：Flink（实时计算）、Spark（离线计算）、Hive（数据仓库）
- **数据同步与调度**：Canal/DataX（数据采集同步）、DolphinScheduler（任务调度）

**模块划分**：
1. **前端模块**：用户端、商家端、骑手端、管理后台的页面交互
2. **后端模块**：业务接口、权限控制、订单处理等核心逻辑
3. **大数据计算工程**：实时指标计算、离线数据统计、报表生成
4. **数据同步和调度工程**：业务库到数据仓库的增量/全量同步，计算任务定时调度

## 3. 功能需求

### 3.1 前端模块（Vue + Axios）

#### 3.1.1 用户端
- 注册/登录（手机号、密码，支持短信验证）
- 基于位置的商家列表（按距离、评分、销量排序）
- 商家详情（菜品分类、菜品信息、评价）
- 购物车管理（增减菜品、清空）
- 下单（选择收货地址、支付方式）
- 订单状态追踪（已支付、商家接单、骑手取餐、配送中、已送达）
- 个人中心（地址管理、订单历史、评价）

#### 3.1.2 商家端
- 登录/注册（商家资质审核）
- 菜品管理（添加、修改、上下架，设置价格、图片）
- 订单管理（接收订单、备餐、出餐状态更新）
- 营业数据概览（当日订单数、营业额）
- 评价查看与回复

#### 3.1.3 骑手端
- 骑手注册、在线状态切换
- 新订单推送、抢单/系统派单
- 导航到商家、取餐确认、导航到用户、送达确认
- 骑手收益统计（当日、历史）

#### 3.1.4 管理后台
- 用户管理（列表、封禁/解封）
- 商家管理（审核、信息修改、营业状态控制）
- 骑手管理（审核、信息、状态）
- 订单管理（订单查询、异常处理、退款审核）
- 数据报表展示（基于大数据计算结果的可视化图表）

### 3.2 后端模块（Spring Boot + MyBatis + MySQL）

- 用户服务：注册、登录、鉴权（JWT）、地址管理
- 商家服务：菜品CRUD、订单处理、评价管理
- 骑手服务：状态管理、接单逻辑、定位跟踪
- 订单服务：下单、支付回调、状态流转、退款
- 搜索/推荐：基于简单规则的商家排序（后期可接入算法）
- 消息推送：订单状态变更通知（可结合WebSocket）
- 权限控制：基于角色的访问控制（RBAC）

**数据库设计核心表**：
- `user`、`address`
- `merchant`、`menu`、`menu_item`
- `rider`
- `order`、`order_item`、`order_status_log`
- `payment`、`evaluation`

### 3.3 大数据计算工程（Flink + Spark + Hive）

- **实时计算（Flink）**
    - 实时订单量、实时GMV统计（按分钟/秒聚合，大屏展示）
    - 热门菜品TopN（滑动窗口）
    - 骑手实时位置与配送监控
    - 订单异常实时告警（长时间未接单等）

- **离线计算（Spark）**
    - 日/周/月订单汇总统计
    - 用户行为分析（复购率、留存率）
    - 商家销量排行、菜品热度排行
    - 骑手绩效统计（配送时效、完成率）

- **数据仓库（Hive）**
    - 构建分层数据模型（ODS → DWD → DWS → ADS）
    - 存储历史订单、用户行为、商家、骑手全量数据
    - 支持管理后台报表查询与自助分析

### 3.4 数据同步和调度工程（Canal/DataX + DolphinScheduler）

- **数据同步**
    - 使用 Canal 监听 MySQL binlog，实时同步增量数据到 Kafka，再由 Flink 消费写入 Hive/Kafka
    - 使用 DataX 实现 MySQL 历史全量数据到 Hive/HDFS 的离线同步
    - 同步任务管理：全量初始化、增量续传、异常重跑

- **任务调度（DolphinScheduler）**
    - 调度 DataX 全量/增量同步任务
    - 调度 Spark 离线计算任务（每日凌晨执行）
    - 调度 Hive 数据仓库层级转换脚本
    - 任务依赖配置、失败重试、告警通知

## 4. 非功能需求

- **性能**：核心接口响应时间 < 200ms，支持并发 1000+ 用户同时下单
- **可用性**：服务95%可用，关键环节有降级方案
- **数据一致性**：订单状态变更要求最终一致性，支付与订单采用可靠消息机制
- **安全性**：接口防篡改、防SQL注入、XSS，用户数据加密存储
- **可扩展性**：微服务化设计，各服务可独立扩展

## 5. 项目里程碑规划

| 阶段 | 任务内容 | 涉及模块 |
|------|---------|----------|
| 第一阶段 | 搭建后端框架、用户注册登录、商家菜品管理基础接口 | 后端 |
| 第二阶段 | 前端用户端核心页面开发，与后端联调下单流程 | 前端+后端 |
| 第三阶段 | 商家端、骑手端、管理后台开发 | 前端 |
| 第四阶段 | 数据同步搭建（Canal+DataX），数仓建模 | 数据同步 |
| 第五阶段 | Flink实时指标、Spark离线计算开发，调度配置 | 大数据计算+调度 |
| 第六阶段 | 全链路测试、性能优化、上线部署 | 全部 |
